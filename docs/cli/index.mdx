# CLI reference

The `jwtop` CLI provides commands covering the full JWT workflow, including security testing.

| Command | Description |
|---|---|
| [`decode`](./decode) | Decode and pretty-print a token without verifying its signature |
| [`verify`](./verify) | Verify a token's signature and standard claims |
| [`create`](./create) | Create and sign a new token from scratch |
| [`sign`](./sign) | Re-sign an existing token with a different algorithm or key |
| [`crack`](./crack) | Probe a server for JWT vulnerabilities using all known exploits |
| [`exploit`](./exploit) | Produce a modified token with a specific security exploit applied |

## Global usage

```
jwtop <command> [flags]
jwtop --help
jwtop version
```

All commands exit `0` on success and `1` on error. Verification failure also exits `1`.

## Global flags

| Flag | Description |
|---|---|
| `--sqa-opt-out` | Opt out of anonymous usage statistics and crash reporting |
